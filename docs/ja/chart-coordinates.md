# 編み図座標系の約束

> Source: [English version](../en/chart-coordinates.md)

このドキュメントは構造化編み図データモデル (ADR-008) が使用する座標系を確定する。レンダラ、エディタ、インポータ、エクスポータは全てこの約束を共有する。

## 矩形グリッド (`RECT_GRID`)

- `x` は **右** が正。`x = 0` は一番左の列
- `y` は **上** が正。`y = 0` は一番下の段 (1 段目)
- 原点 `(0, 0)` は左下のセル

編み図は慣習的に下から上に読む (最初に編む段がチャートの下端)。本システムの座標系もこの読み方向に合わせているため、セル座標 `(x = 6, y = 2)` が編み手の言う「3 段目の 7 目」と一致する。

### レンダリングに関する注記

画面座標は通常 `y` が下方向に増える。Phase 31 の Compose / SwiftUI レンダラはビューポート層で `scaleY(-1)` (+ チャート高さ分の平行移動) を適用して、チャート座標→画面座標への変換を行う。ドメインモデルを使う側 (リポジトリ、ユースケース、インポータ) は常にチャート座標で操作する — 反転は描画呼び出しの場所でだけ起こる。

### マルチセルシンボル

`width > 1` または `height > 1` のセルは矩形 `[x, x + width) × [y, y + height)` を占有する。アンカー `(x, y)` から右方向と上方向に広がる。

`(x = 3, y = 5)` の 2×2 cable cross は列 3,4 と段 5,6 をカバーする。

## 輪編み / polar (`POLAR_ROUND`)

- `x` は **その段の中でのステッチインデックス** (0 始まり)。編み地を外側から見た時の時計回りに増加
- `y` は **ring index** (0 始まり)。中心から **外側** に増加
- `y = 0` は最内周 (作り目 / 1 段目)
- `ChartExtents.Polar.stitchesPerRing[y]` がその段のステッチ数。`x` はこの値で mod を取る

### レンダリングに関する注記

Phase 31 の polar レンダラは `(x, y)` を `angle = 2π · x / stitchesPerRing[y]`, `radius = baseRadius + y · ringSpacing` で画面にマップする。角度の増加方向 (時計回り / 反時計回り) はレンダラの判断事項であり、データモデル自体は方向に依存しない。

### マルチセルシンボル

polar モードのマルチセルシンボルは **Phase 35 (editor advanced)** に繰り越し。Phase 29 では polar チャートは `width = height = 1` のみ受け入れる。ドメインモデルは `width` / `height` を常に持つためスキーマ変更は発生しない。

## 回転

`ChartCell.rotation` は度数、値は `0`, `90`, `180`, `270`。チャート座標系における時計回り (= `scaleY(-1)` 後の画面座標で反時計回り)。

軸非整列の回転は v1 では意図的に非サポート。

## デフォルト値

| フィールド | デフォルト |
|---|---|
| `width` | `1` |
| `height` | `1` |
| `rotation` | `0` |
| `colorId` | `null` |
| `ChartLayer.visible` | `true` |
| `ChartLayer.locked` | `false` |

## バウンディングボックス

`ChartExtents.Rect` は両端 inclusive: `minX ≤ x ≤ maxX`, `minY ≤ y ≤ maxY`。空のチャートは `minX = 0, maxX = -1` (正規な空範囲) とする — `null` にはしない。これにより extents は常に well-formed な `Rect` となる。
