## 1. 判定

| グリフ | 判定 |
|---|---|
| `jis.crochet.reverse-sc` | **ship-with-followup** — 矢印形 vs. 塗り矢印頭はユーザ好み確認として follow-up に escalate、シェブロン形は防衛可能で描画も清潔 |
| `jis.crochet.puff` | **ship** — JIS L 0201＋ヴォーグ社定番形に一致 |
| `jis.crochet.hdc-cluster-3` | **ship** — `dc-cluster-3` を下地に斜線を省略（hdc 高さを示す JIS シグナル） |

ブロッカーなし。28 グリフで安全にマージ可。

## 2. 幾何検証（24dp 視認性、既存グリフとの衝突）

- **`reverse-sc`**: `×` を y∈[0.15, 0.6] に圧縮、左向き対称 V を y∈[0.77, 0.97] に配置（中心 y=0.87）。Phase 30.3 の小修正（頂点 x=0.18、端点 (0.35, 0.77)/(0.35, 0.97)）により、V はチェックマークではなく矢印頭として読める。セル端との ~3% マージンで、stroke cap の clip をどのレンダラでも回避。`sc`（中央 `×`、マーカー無し）、`sc-fl`/`sc-bl`（`×` + 下方／上方開口アーク）と衝突せず。
- **`puff`**: 3 本の収束ステムが y=0.20 で `x∈[0.25, 0.75]` の開いた上端バーに到達。ステム先端（0.35 / 0.5 / 0.65, y=0.20）がバーに接するため、24dp でも束をキャップが閉じて見える（浮かない）。`dc-cluster-3`（平行ステム + 閉じた楕円 + 斜線）、`popcorn`（塗りつぶし五角形）と明確に区別可能。
- **`hdc-cluster-3`**: 平行ステム + 閉じた楕円、斜線なし — hdc 高さを示す JIS シグナル。`dc-cluster-3` との差異は斜線の有無のみ。24dp では約 10px の差で、やや詰まっているが識別可能。§5 参照。

## 3. JIS／JP 出版社権威

- **`reverse-sc`**（逆細編み）: JIS L 0201 は専用グリフを規定せず。日本ヴォーグ（毛糸だま / Let's Knit）・文化出版局ともに `×` に方向マーカーを重ねる運用で、シェブロン形は防衛可能。近年のヴォーグ一部では塗り矢印頭を採用する例あり — follow-up としてユーザ好みに escalate、ブロッカーではない。
- **`puff`**（パフ編み）: JIS L 0201 表 2 + 毛糸だま定番形に合致（縦束 + 上部一本キャップ、楕円なし）。あらゆる国内リファレンス本に対して防衛可能。
- **`hdc-cluster-3`**（中長編み 3 目の玉編み）: ヴォーグ／雄鶏社規約に一致。JIS は玉編みファミリを示すが、hdc 派生は必ずしも網羅していない。それでも「楕円キャップ + 斜線なし ＝ hdc 束」と正しく読める。

## 4. カバレッジ所感

これで `jis.crochet.*` は国内商業クロシェパターンの約 90% を覆う。Phase 30.4 以降の優先順位で**まだ欠落しているもの**:

1. **`hdc-cluster-5`**（中長 5 目玉編み） — `dc-cluster-5` のミラー、`widthUnits=2`。
2. **Solomon's knot / ラブノット** — 透かし編みショールで頻出。
3. **交差長編み (Crossed dc)** — 2 本の dc が X 字に交差。アラン風クロシェで使用。
4. **ブリオンステッチ** — 渦巻きマーク。ニッチだが毛糸だまドイリー号に出現。
5. **長いピコット**（`picot-4`、`picot-5`）— パラメトリック `picot-N` として整理する方がきれい。

いずれも Phase 32（エディタ MVP）の障害にはならない。エディタ使用テレメトリを見ながら機会的に追加。

## 5. 幾何の微修正と実装後調整

Phase 30.3 PR 内で反映:

- **`reverse-sc` シェブロン** — `M 0.35 0.75 L 0.15 0.88 L 0.35 0.88`（底辺が水平でチェックマークに見えた非対称 V）から `M 0.35 0.77 L 0.18 0.87 L 0.35 0.97`（y=0.87 中心の対称 V、矢印頭ポインタ、セル端から 3% マージンで stroke cap clip 回避）へ変更。
- **`puff` 上端バーとステム** — バーを `0.28→0.72` から `0.25→0.75` へ拡幅、3 本のステム先端を y=0.22/0.25/0.25 から y=0.20 に持ち上げてバーに接続。JIS の「一つの高さに引き揃える」記述に整合し、24dp の低 DPI Android でステムとバーが切れて見える問題を解消。
- **`hdc-cluster-3` のエイリアス** — `hdc3tog-puff` エイリアスを削除。`puff` 単語検索で `jis.crochet.puff` ではなく本グリフに偽陽性ヒットするのを防ぐため。`bob-hdc` は継続。

Phase 30.4（`hdc-cluster-5` と同時施策）へ繰り延べ:

- **`hdc-cluster-3` ステム間隔** — 斜線がないため、低 DPI Android 端末 24dp では 3 本平行ステムが 1 本の太線に見えうる。外側ステムを `0.25 / 0.75` から `0.22 / 0.78` へ移動し、間隔を ~6% 広げる案。純粋に審美的修正、hdc-cluster-5 設計と合わせて再評価。
