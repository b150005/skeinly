## 1. サマリ

**Phase 30.2-fix を挟んでからマージ推奨** — 大半のグリフは許容範囲だが、商業パターン文脈で誤読される工芸的正確性の問題が 2 件：

- **`sl-st`**：JIS / CYC / 主要日本出版社のいずれも塗り潰しの黒丸。現状は枠線のみの楕円。
- **`dc2tog` / `dc3tog`**：上部の短い横線が 24dp で玉編みの閉じた楕円と視覚的にほぼ同一。減目と玉編みの区別はクロシェ記号セット最大の視覚的差分であり、ここが曖昧なのは致命的。

それ以外は細かい幾何修正・参照規約の選択・ラベル微調整のいずれか。25 グリフは Phase 30.2 目標（25–35）を満たすが、商業パターンで高頻度な数点が未搭載であり Phase 30.3 に送り込みたい。

## 3. 横断的課題

### 3.1 `sl-st` の塗り vs. 枠線（描画器起因の問題）

JIS L 0201 表 2・CYC・日本ヴォーグ・文化出版局ともに、引き抜き編みは**塗り潰しの黒丸／楕円**（セル幅の 0.3〜0.5 程度）で描く。現描画器は枠線のみを描き、「塗り潰し」を表現する手段が無い。その結果、どれほど `sl-st` を小さく枠線で描いても、日本の編み手には sl-st と認識されないドーナツに見える。

解決策は 2 系統、いずれも Phase 30.2-fix スコープ：

1. **描画器側**：`SymbolDefinition` に `fill: Boolean`（または `SymbolStyle` enum）を追加し、Compose と SwiftUI の描画器両方で尊重する。範囲は小さく、長期的に正しい。塗り潰しを要する将来の JIS グリフ（塗り三角の sl-st 異体、塗り菱形ボブル異体など）への再利用も効く。
2. **パス技巧**：楕円を二重描画して内側を塗り潰しに見せる。ズーム拡大で破綻し、将来に悪い前例を残す。

強く推奨：選択肢 1。カタログ横断の課題で再発が見込まれる。

### 3.2 減目 vs. 玉編みの区別

`dc2tog` / `dc3tog`（減目）vs. `dc-cluster-3` / `dc-cluster-5`（玉編み）は記号セット中で最も視覚的に類似した対であり、24dp + 2px stroke では：

- **減目**：上部の短い水平開放線（`M 0.4 0.18 L 0.6 0.18`）
- **玉編み**：上部の閉じた楕円（cubic-bezier）

の差は**正味約 1 ピクセルの空隙／面積**に過ぎない。JIS は減目の上部を**外側斜線より長く明確に突き出した開放水平線**で描き（「開いている」信号が低ズームでも残る）、玉編み楕円は**丸く閉じて**非曖昧に描く。dc2tog/dc3tog の上部線を伸ばし、場合によっては僅かに内向き弧にして「開放」信号を強化すること。

### 3.3 ポスト系引き上げ鉤方向は出版社差異

`fpdc` が右、`bpdc` が左、という方向は CYC と一部米国出版社の慣例。日本ヴォーグは**逆慣例**を採用（表＝手前側に下向きカーブ、裏＝奥側に上向きカーブ）。JIS は沈黙。どちらも許容可だが、一つを選んで `jaDescription` に明記し、JA ユーザーの混乱を避けること。§5 参照。

### 3.4 `turning-ch` の数固定

`turning-ch` は 3 つの鎖楕円で固定され、暗黙的に dc 段前提になっている。商業日本クロシェパターンでは、立ち上がり鎖の数は最初の編み目高さに従う（hdc=2, dc=3, tr=4, dtr=5, trtr=6）。対応策 3 つ：

- 編み目別バリアント（`turning-ch-2`, `turning-ch-3`, ...）を追加：機械的で最小コード変更。
- `turning-ch` に `count` スロットを付与し動的描画：描画器変更必要、スコープ拡大。
- 数を描かない抽象的「立ち上がり束」グリフ 1 種で済ませる：最安だが情報量薄い。

推奨：Phase 30.3 で 1 つ目（既存カタログ慣例 — `dc-cluster-3` vs `dc-cluster-5` と同じ様式 — に整合、描画器変更不要）。

### 3.5 24dp でのストローク密度

`qtr` は 4 本の斜線を `y=0.22 – 0.70`（0.12 間隔）に詰める。24dp では斜線中心間が約 2.9px、stroke 幅 2px を差し引くと約 1px の隙間。縮小または低 DPI では一塊に潰れる。`dtr`（3 本）は境界、`tr`（2 本）は安全。描画器側で stroke 幅テーパー処理または最小セルサイズガードを検討 — 記号定義側の修正ではない。

## 4. カバレッジ評価

25 グリフは Phase 30.2 目標（25–35）達成。バイアスは**基本編み目と構造的減目**に寄っており、v1 としては正しい優先順位。ただし、以下の欠落は商業出版社が実際の日本クロシェパターンを代替無しで組版するのを阻む：

| 未搭載 | 頻度 | 重要性 | 推奨フェーズ |
|---|---|---|---|
| **逆細編み（crab stitch）** | 高（縁編み、ほぼ全ての仕上がり作品） | カーディガン／ブランケット縁の定番。左→右方向という独自性。 | 30.3 — 最優先 |
| **パフステッチ** | 中〜高（アミグルミ、レース、ベビー） | JIS で popcorn・玉編みと別グリフ。 | 30.3 |
| **中長編み玉編み（bob-hdc）** | 中（レース、スロー） | `dc-cluster-3` と同様の役割だが hdc 線。シェル模様で多用。 | 30.3 |
| 伸ばす細編み（exsc）／ロング sc | 中（アミグルミ） | CYC 標準、JIS 沈黙。 | 30.3 or 保留 |
| ファウンデーション sc/dc | 中（増加傾向） | 米式慣例、鎖無しスタート。 | 30.4 or 保留 |
| スパイクステッチ（ロングループ類） | 低〜中 | カラーワーククロシェ。 | 保留 |
| 段を輪にする sl st（別グリフ化する場合） | 高 | `sl-st` 共用なら追加不要；別グリフなら追加。 | 30.3 要判断 |
| 玉編み減目（cl3tog 等） | 低 | レース以外では希少。 | 保留 |

**Phase 30.3 最優先 3 件：** 逆細編み、パフステッチ、中長編み玉編み。これで Phase 30.2 が埋め残した日本商業クロシェ組版ニーズの約 9 割をカバーできる。

## 5. ユーザーへの未解決論点

Knitter が解決しきれず、ユーザー判断を仰ぐ項目。

1. **`sl-st` 塗り潰し — `SymbolDefinition` に `fill: Boolean`（または `SymbolStyle`）を追加し両描画器に尊重させるか、v1 はパス技巧の近似で済ませるか？** 推奨は追加（小スコープ、将来保険。塗り三角 sl-st 異体・塗り菱形ボブル異体等も欲しくなる）。
   - **Team（Phase 30.2-fix）：** `SymbolDefinition` に `fill: Boolean = false` を追加。Compose 側 `SymbolDrawing.drawSymbolPath` は `Fill` / `Stroke` を切替え、SwiftUI 側の `StructuredChartViewerScreen` と `SymbolGalleryScreen` は `context.fill` / `context.stroke` を切替える。本 PR で `fill = true` に切替えるのは `jis.crochet.sl-st` のみ。将来の塗り三角・ボブル類は同 field を opt-in して取り込める。
2. **`qtr` 命名 — 米式 "quadruple treble"（id `qtr` と JA `四つ巻長編み` に整合）か、英式 "triple treble"（現 `enLabel` + `cycName="trtr"` に整合）か？** 現状は内部不整合。id が既に `qtr`、JIS ・日本ヴォーグも 四つ巻 = quad tr と表記するため米式推奨。
   - **Phase 30.2 内で解決（inline）：** code-reviewer の推奨を受けて米式を採用。`enLabel` → `"Quadruple treble crochet (qtr)"`、`cycName` → `"qtr"`、`aliases = listOf("trtr", "triple treble")` を登録し、英式表記からも同一 glyph を引けるようにした。geometry は未変更。
3. **`sc` 横線 — 中央 `+`（CYC 英語 + 一部日本出版社）か、斜め `×`（JIS L 0201 + 多数日本出版社）か？** 工芸的正誤ではなく JA 読者向け権威性の判断。JP を主オーディエンスとするなら `×` がやや本格的。延期可。
   - **Team（Phase 30.2-fix）：** JIS `×` を採用。ADR-008 §6 + Phase 30.5 追補により、`jis.*` namespace では JIS と CYC が disagree した場合 JIS 側に倒す方針。日本ヴォーグ・文化出版局も斜め十字を採る。`pathData` を `M 0.3 0.3 L 0.7 0.7 M 0.7 0.3 L 0.3 0.7` に更新。
4. **`dc2tog` / `dc3tog` 上部線 vs. 玉編み楕円 — 減目の上部線を広げ外側斜線より長くするか（推奨）、ズームで区別頼みか？** 本セットで最大の視認性バグ、修正推奨。
   - **Team（Phase 30.2-fix）：** 広げる。`dc2tog` 上部線を `0.4→0.6` から `0.15→0.85` に、`dc3tog` 上部線を `0.4→0.6` から `0.10→0.90` に拡大。外側斜線の端点を visibly 越え、24dp で「開いた水平線」として読めるようになり cluster の閉じた楕円と区別がつく。
5. **sc-dec 横線の高さ — 現 `y=0.55`（中央）のままか、JIS 通り頂点付近 `y≈0.25` に移すか？** 中央は CYC 風、頂点は JIS 準拠。どちらも工芸的非難無し。
   - **Team（Phase 30.2-fix）：** JIS に従い頂点寄せ。`sc2tog` 横線 `y=0.55` → `y=0.32`（頂点 `y=0.25` の直下）、`sc3tog` 横線 `y=0.55` → `y=0.28`。横線が収束する腕の内部に収まり、JIS sc-cross 署名が頂点付近に残る。
6. **ポスト系引き上げ鉤方向 — CYC 慣例（表 = 右・裏 = 左）維持、あるいは日本ヴォーグ慣例（表 = 下カーブ・裏 = 上カーブ）に切替？** 工芸バグではないが、一つ選んで明文化。
   - **Team（Phase 30.2-fix）：** 方向は CYC（表 = 右、裏 = 左）を維持。JIS は方向に silent、両慣例とも accepted のため。別途、底部 L 字を二次曲線 C 巻きに置換し「ポストを巻く鉤」として読めるようにする（旧 L 字は矢印に見えていた）。`fpdc` 底部は `M 0.5 0.8 Q 0.85 0.8 0.85 0.95`、`bpdc` は `Q 0.15 0.8 0.15 0.95` で mirror。
7. **Popcorn 描画 — CYC 風放射円のままか、JIS 風五角形／束＋冠に切替？** JP 主オーディエンスなら JIS 形がより本格的。軽微。
   - **Team（Phase 30.2-fix）：** JIS 五角形（頂点上）に切替。Q3 と同じ ADR-008 namespace authority 論理。頂点を `(0.5, 0.15)`、`(0.79, 0.36)`、`(0.68, 0.7)`、`(0.32, 0.7)`、`(0.21, 0.36)` に置く閉じた五角形を stroke 描画（fill ではない）。cell 内に収まり、cell 幅一杯に広がる cluster 楕円とは混同しない。
8. **`turning-ch` の数 — Phase 30.3 で編み目別バリアント（`turning-ch-2, -3, -4, -5, -6`）を追加、あるいは今すぐ `count` スロットでパラメータ化？** 個別バリアント案は既存カタログ慣例（例: `dc-cluster-3` vs `dc-cluster-5`）と整合し描画器変更不要、パラメータ化は長期的に綺麗だがスコープ拡大。
   - **Team（Phase 30.2-fix）：** Phase 30.3 で per-stitch variants を追加する方針で deferral。`dc-cluster-3 / -5` の per-count 慣例と一致し描画器変更不要、editor（Phase 32）の UX が固まる前に parameter-slot 契約を決め打ちしないで済む。現行の `turning-ch`（3 楕円 = dc default）はそのまま stop-gap として残置。
9. **`inc-sc` / `inc-dc` 幾何 — V 字拡散（現）維持、JIS 流「2 つのグリフを底部ブラケットで連結」形に変更？** V 字は西洋、ブラケットは JIS。軽微な権威性判断。
   - **Team（Phase 30.2-fix）：** V 字を維持。JIS は composite glyph について silent。V 字は increase として明白に読め、最も一般的な Western chart rendering と一致する。JP 出版社からの混同 feedback があれば bracket 形を再評価する。
10. **Phase 30.3 スコープ — 逆細編み・パフ・中長編み玉編みを最優先 3 件で確定？ それともファウンデーション系や伸ばす細編みを先に？** §4 の最優先 3 件を推奨。
    - **Team（Phase 30.2-fix）：** 確定 — 逆細編み・パフステッチ・中長編み玉編みを Phase 30.3 top-3 とする。ファウンデーション系・伸ばす細編みは §4 頻度表に従い Phase 30.4 / 保留へ。
