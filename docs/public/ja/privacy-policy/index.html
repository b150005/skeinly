プライバシーポリシー

Skeinly — 施行日: 2026年4月15日

概要

Skeinly（以下「本アプリ」）は、Skeinlyチームが開発した編み物パターン管理アプリケーションです。本プライバシーポリシーでは、本アプリをご利用いただく際の情報の収集、使用、および保護について説明します。

収集する情報

アカウント情報

アカウント作成時に以下の情報を収集します：

  • メールアドレス — 認証およびアカウント復旧に使用
  • 表示名 — プロフィールおよび共有コンテンツに表示（任意）

サードパーティプロバイダ（Google、Appleなど）を使用してサインインする場合、認証に必要な情報（メールアドレスと表示名）のみをプロバイダの許可に基づいて受信します。

ユーザー作成コンテンツ

本アプリは、お客様が作成する以下のコンテンツを保存します：

  • 編み物プロジェクト（タイトル、段数、ステータス、メモ）
  • 編み図パターン（名前、ゲージ、糸情報、針サイズ、チャート画像）
  • プロジェクトメモに添付された進捗写真
  • 共有パターンおよびプロジェクトへのコメント
  • アクティビティフィードのエントリ

自動的に収集される情報

本番リリース版の本アプリでは、アナリティクス、クラッシュレポート、トラッキング機能は一切有効化されません。デバイスや利用状況のテレメトリを能動的に収集することはありません。一部のサードパーティ SDK（例: PostHog、Sentry）はバイナリにリンクされていますが、本番ビルドでは一切初期化されません — ネットワーク接続は開かれず、データも収集されません。

以下の情報は収集しません：

  • 位置情報
  • 連絡先またはアドレス帳データ
  • ブラウジング履歴
  • 広告識別子
  • アナリティクスまたはトラッキングデータ

診断データ（ベータビルドのみ）

Skeinly のベータビルド（TestFlight Internal または Google Play Internal Testing 経由で配布）を利用しており、かつ「設定 → ベータ → 診断データの送信」で明示的にオプトインされている場合、本アプリは以下のデータを PostHog（PostHog のプライバシーポリシーを参照）に送信します：

  • アプリのバージョンとビルド番号
  • OS バージョン
  • デバイス機種
  • ロケール（例：en-US、ja-JP）
  • 匿名のデバイス識別子（PostHog SDK が生成するランダムな UUID。アカウント、メールアドレス、その他アプリのローカルストレージ外の永続的識別子とは紐付きません）
  • 訪問した画面の名前（例：「ProjectList」「Settings」）
  • カテゴリカルなアクション名（例：「create_project」「fork」）

オプトインしている場合でも、以下の情報は絶対に送信しません：

  • パターン名、プロジェクトタイトル、メモ本文
  • 検索クエリ、コメント本文、その他お客様が入力した自由記述テキスト
  • 写真や画像
  • メールアドレスや表示名

アプリ内バグレポーター（「フィードバックを送信」ジェスチャーまたは設定からのエントリ）でバグ報告を送信した場合、レポートはサーバーサイドプロキシ（submit-bug-report という名前の Supabase Edge Function）に送られ、プロキシがお客様に代わって公開リポジトリ b150005/skeinly に GitHub Issue を作成します。プロキシは「Skeinly Feedback」という GitHub App として動作し、その権限はこの 1 リポジトリの Issue 作成・更新のみに限定されています（コードの閲覧、他リポジトリへの変更、Skeinly アカウントへの Issue 管理以外の操作はできません）。Issue 本文には、お客様が入力した説明文、アプリバージョン、OS、デバイスモデル、ロケール、匿名の PostHog デバイス識別子（調査時に PostHog ダッシュボードのデータと相関させるため）、および直近 10 件のアプリ内操作履歴が含まれます。リポジトリは公開のため、Issue のタイトルと本文は誰でも閲覧できます。送信時のメタデータ（IP アドレス、リクエスト時刻）は Supabase からは見えますが、プロキシ側で永続化はされません。GitHub Issue は「Skeinly Feedback」GitHub App の名前で作成され、お客様個人の GitHub アカウントの名前にはなりません。同じデバイスで GitHub にログインしていても、レポートにお客様の GitHub アイデンティティは紐付きません。

同意は「設定 → ベータ → 診断データの送信」でいつでも撤回できます。撤回は即座に反映され、その後の収集を停止し、ローカルにキャッシュされたイベントバッファをクリアします。本番リリース版の Skeinly にはこの機能自体が含まれません — SDK はバイナリにはリンクされますが初期化は行いません。

本機能は、GDPR および日本の改正個人情報保護法に準拠して、デフォルトはオフ（オプトイン方式）です。

プッシュ通知

Skeinly は、お客様が所有またはフォークしたパターンに対するコラボレーションイベントについてプッシュ通知を送信します。具体的には次の場合です：所有パターンに対してプルリクエストが作成された、参加中のプルリクエストにコメントが追加された、お客様が作成したプルリクエストがマージまたはクローズされた、所有パターンに対するプルリクエストがクローズされた。

プッシュ通知はオプトイン方式です。OS レベルの通知許可は、アプリ内プロンプト（プルリクエスト関連の操作を初めて行う際に表示）またはデバイスの「設定 → 通知 → Skeinly」画面で明示的に許可していただきます。OS レベルの許可付与が同意となります — 別途のアプリ内アナリティクス様トグルは存在しません。

許可を付与いただいた場合、当方のデータベース（テーブル: device_tokens）に以下を保存します：

  • Apple Push Notification service（iOS）または Firebase Cloud Messaging（Android）が発行する匿名のデバイストークン — これはデバイス・アプリ単位の識別子であり、アプリ再インストールや OS リセット時に OS によりローテーションされます。Apple や Google が保持する OS 側のクロスリファレンスを経由しない限り、お客様の物理デバイスやお客様個人へ直接的に紐づきません
  • デバイスのロケール（BCP-47 タグ、例: en-US または ja-JP） — サーバー側で通知本文をローカライズするために使用します
  • プラットフォーム（ios または android）と Skeinly のユーザー ID — 適切な相手に通知を届けるために使用します

通知本文には、イベントを引き起こした他の Skeinly ユーザーの表示名が含まれます（例：「AliceさんがGranny Squareにプルリクエストを開きました」「BobさんがAdd bobble rowにコメントしました」）。これは受信者であるお客様がロック画面で目にする他の Skeinly ユーザーに関するデータであり、お客様自身のデータではありません。同様に、お客様の操作に関する通知を受信する他のユーザーは、お客様の表示名を目にすることになります。

プッシュ通知の許可は、デバイスの「設定 → 通知 → Skeinly」でいつでも撤回できます。撤回は OS レベルで行われ、即座に反映されます。Skeinly アカウントを削除（「設定 → アカウント → アカウントを削除」）された場合、外部キー制約により device_tokens 行はカスケード削除され、以降の通知送信は即座に停止します。

プッシュ通知の配信には、iOS では Apple Push Notification service（Apple のサーバー、Apple のプライバシーポリシーを参照）、Android では Firebase Cloud Messaging（Google のサーバー、Google のプライバシーポリシーを参照）を使用しています。Apple および Google が運営するルーティングインフラの管理権限・アクセス権限は当方にはありません — 当方は不透明なデバイストークンと通知ペイロードを Apple / Google に渡し、それらの会社がお客様のデバイスに配信します。

情報の使用目的

  • サービスの提供: プロジェクト、パターン、進捗をデバイス間でアクセスできるよう保存
  • 共有機能: お客様が選択した場合に、他のユーザーとパターンやプロジェクトを共有
  • アカウント管理: お客様の身元認証とアカウント管理
  • アプリの改善: 技術的な問題の診断と信頼性の向上

データの保存とセキュリティ

お客様のデータはSupabaseを使用して安全に保存されます：

  • 暗号化されたデータ通信（TLS/HTTPS）
  • Row-Level Security（RLS）ポリシーにより、お客様自身のデータのみにアクセス可能
  • 署名付きURLによる画像の安全なファイルストレージ

本アプリはオフライン使用にも対応しています。オフライン時に作成されたデータはデバイス上にローカル保存され、接続が回復した際に同期されます。

データの共有

お客様の個人情報を広告やマーケティング目的で第三者に販売、貸与、または共有することはありません。

お客様のコンテンツは、パターンやプロジェクトを明示的に共有することを選択した場合のみ、他のユーザーに表示されます。

お客様の権利

お客様には以下の権利があります：

  • アクセス — アプリを通じていつでもデータにアクセス
  • 更新 — 設定画面からプロフィール情報を更新
  • 削除 — 設定画面からアカウントおよび関連する全データを削除（設定 > アカウント削除）
  • エクスポート — 以下のメールアドレスにご連絡ください

アカウント削除は恒久的であり、プロジェクト、パターン、進捗、コメント、アップロード画像の全てがサーバーから削除されます。

子供のプライバシー

本アプリは13歳未満の子供を対象としていません。13歳未満の子供から意図的に情報を収集することはありません。子供が個人情報を提供したと思われる場合は、ご連絡ください。

ローカルオンリーモード

本アプリはアカウントを作成せずに使用できます。ローカルオンリーモードでは、全てのデータはデバイス上にのみ保存され、サーバーに送信されることはありません。

データ処理（EU/EEA在住の方）

欧州連合または欧州経済領域にお住まいの場合、以下の追加情報が適用されます：

  • 処理の法的根拠: お客様のデータは、契約の履行（お客様が登録したサービスの提供）および正当な利益（アプリの信頼性向上）に基づいて処理されます。
  • データ処理者: お客様のデータはSupabaseによって保存・処理されます。SupabaseはAWSインフラストラクチャを使用し、当社に代わってデータ処理者として機能します。
  • データ保持: お客様のデータはアカウントが有効な限り保持されます。アカウント削除時に、関連する全データがサーバーから恒久的に削除されます。
  • GDPRに基づくお客様の権利: 上記の権利に加えて、データポータビリティの権利、処理制限の権利、および現地のデータ保護当局に苦情を申し立てる権利があります。

ポリシーの変更

本プライバシーポリシーは随時更新される場合があります。変更は上記の「最終更新」日に反映されます。変更後も本アプリを継続して使用することは、更新されたポリシーの承認とみなされます。
