## Scope

Phase 40 GA release-prep bullet from `tech-debt.md` → "Bug-report Settings entry
GA opening". Removes the `BuildFlags.isBeta` gate on the "Send Feedback" row in
the Settings screen and promotes that row out of the Beta section into the
About / Legal section so every GA user (not just Beta testers) can file a
structured bug report from Settings. The row's click handler is routed through
the ViewModel via a new `SettingsEvent.SendFeedbackTapped` (no analytics
emitted today; symmetric with `SubscribeToProTapped`).

**Out of scope** (deliberate): the diagnostic-data toggle and Notifications row
stay in the Beta section. The Android 3-finger-long-press and iOS shake
gestures stay `BuildFlags.isBeta && analyticsOptIn`-gated as tester shortcuts.
No `ClickActionId.SendFeedback` is added (would require editing the analytics
enum file outside the declared write-set; analytics breadcrumb deferred).

## Result summary

Phase 40 GA "Bug-report Settings entry GA opening" landed. The "Send Feedback"
row is now in the About / Legal section (alongside Contact Support), with no
`BuildFlags.isBeta` gate — every GA user can file a structured bug report
from Settings. Click handling routes through a new `SettingsEvent.SendFeedbackTapped`
event (no-op handler, no analytics emitted today — symmetric with
`SubscribeToProTapped`). Beta section now holds only the diagnostic-data toggle
and Notifications row. Android 3-finger-long-press and iOS shake gestures
remain `BuildFlags.isBeta && analyticsOptIn`-double-gated as tester shortcuts
(unchanged).

Review findings: code-reviewer APPROVE (0 CRITICAL / 0 HIGH / 1 MEDIUM /

is idiomatic"). security-reviewer APPROVE (0 CRITICAL / 0 HIGH / 1 LOW —
