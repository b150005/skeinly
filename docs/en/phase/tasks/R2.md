**Key design decisions**:
- Cell-level semantics (Compose) / `accessibilityElement(children: .ignore)`
  (SwiftUI) over inner-Icon labels — single SR element per palette cell /
  thumbnail avoids double-announcement of the SF Symbol name + the
  semantic label.
- Reused 4 existing keys (`action_more_options`, `action_sort`,
  `action_undo`, `action_redo`) on iOS — no duplication.
- Single `a11y_label_palette_cell` template (`"Symbol: %1$s"` / `"記号:
  %1$s"`) parameterized with locale-appropriate `def.{ja,en}Label` —
  matches `ChartViewerScreen.kt:747` precedent, avoiding a new model API.
- `paletteSymbolAccessibilityLabel(for:)` helper added to
  `ChartEditorScreen.swift` (private file-scope func) — colocates the
  locale-resolution logic with both PaletteSymbolCell variants without
  bleeding into the SymbolDefinition model.
- LockedPaletteSymbolCell label is `<title_locked_symbol> · <Symbol: name>`
  not just `<title_locked_symbol>` — SR now announces *which* symbol is
  locked, not just that *some* symbol is.
