## Result summary

Centralized the locale-aware symbol-label resolution that the X3 wave intentionally left out of scope. The predicate `locale.startsWith("ja", ignoreCase = true)` was inlined at 5 sites across 4 Kotlin chart-UI files; Y3 extracts it into `SymbolDefinition.localizedLabel(locale: String)` and `SymbolCategory.localizedLabel(locale: String)` extensions under `domain/symbol/LocalizedLabel.kt`. Each call site is now `catalog.get(id)?.localizedLabel(deviceContext.locale) ?: id` (for the 3 catalog-lookup sites) or `def.localizedLabel(deviceContext.locale)` (for the 2 SymbolPaletteStrip sites that already hold a `SymbolDefinition` from iteration). The X3-follow-up KDoc comments at each former ternary site are removed (the follow-up is fulfilled by this commit); the R2 a11y comments at the SymbolPaletteStrip sites are preserved.

**Naming deviation**: Worker prompt suggested `SymbolCatalog.localizedLabel(locale: Locale)`. Implementation diverges to **per-instance extensions on `SymbolDefinition` / `SymbolCategory`** + `locale: String` (not `Locale`). Rationale:

1. 3 sites already had `catalog.get(id)?.let { ... }` shape → calling `.localizedLabel(...)` on the returned `SymbolDefinition` is the most idiomatic substitution.
2. 2 sites in `SymbolPaletteStrip` already hold `def: SymbolDefinition` from iteration; a `catalog.localizedLabel(id, locale)` wrapper would force a redundant map lookup.
3. Adding `localizedLabel` to `SymbolCatalog` interface would require updating 3 existing implementations (`DefaultSymbolCatalog`, `CompositeSymbolCatalog`, `DefaultSymbolPackCatalog`) — more invasive than zero-touch extensions.
4. `locale: String` matches the existing `DeviceContext.locale: String` field. Introducing a KMP `Locale` wrapper is YAGNI for this consolidation.

**Inventory discrepancy**: `tech-debt.md` X3 surface line reads "10 sites across 8 files". Actual observation: **9 sites / 8 files** (Kotlin 5 + Swift 4). The Worker-prompt grep pattern was Swift-syntax oriented (`isJa ? jaLabel : enLabel` ternary) and produced 0 matches against Kotlin's `if (...) jaLabel else enLabel` form; the inventory was corrected before work began. Orchestrator should confirm whether tech-debt's "10" was a typo or whether a 10th site was missed by both the worker grep and the secondary if-else grep.
