### 知見 (operator + orchestrator 向け)

並列 worktree が同一 host の iOS Simulator (UDID 共有) で xcodebuild test を並走させると、app install / app data の state competition で state-dependent XCUITest (Onboarding / ProjectList / ProjectDetail 等) が壊れる。**worker 並列実行時は per-worktree 独立 simulator (`xcrun simctl create` で UDID 別) を割り当てる、もしくは ci-local 実行を serial 化する** ガイダンスが parallel-worktree workflow protocol に追加されるべき。`make ci-local` 中の simctl ロック取得や `IOS_SIM_DEST` per-worktree override などの対応案あり（tech-debt 候補）。
