**Key design decisions (operator-confirmed via agent-team deliberation 2026-05-19)**:

- **Webhook scope: repo-level** (`b150005/skeinly` Settings → Webhooks). App-level rejected (§A1): would couple ADR-020's outbound-only "Skeinly Feedback" App with the new inbound triage concern.
- **Edge Function: `github-webhook` (Deno, verify_jwt = false)**. ADR-020 §Q4 precedent for verify_jwt = false; real auth is HMAC SHA-256 over raw body bytes.
- **Decision mapping: label-based** (`state-resolved-remove` / `state-resolved-keep` / `state-dismissed`). Comment-keyword parsing rejected (§A2) for typo/case fragility + audit-trail dispersion.
- **SLA column: reuse existing `resolved_at`** from migration 031. New column (`state_resolved_at` hypothesized in the spawning worker prompt) rejected via agent-team deliberation (architect/security-reviewer/code-reviewer/implementer/product-manager voices). Rationale: no schema duplication, no migration needed, monitoring queries unchanged, idempotency clear with `WHERE resolved_at IS NULL`, single source of truth for resolution timestamp.
- **Out of scope**: reporter notification, multi-repo subscription, auto-content-removal chaining, bidirectional sync (Supabase → GitHub auto-close — §A5 deferred), ambiguous-label auto-comment, moderator-pool recusal, ML classifiers.
