audit §3.3 M2 information-density parity gap (Compose-iOS asymmetry) closure。iOS `PatternRow` (PatternLibraryScreen.swift L233-260) に 3 番目の details line (gauge / yarnInfo / needleSize joined by ` • `) を追加し、Compose `PatternCard.buildPatternDetails()` と完全一致する information density に到達。

### 設計上の決定

| 項目 | Compose | iOS 採用 | 根拠 |
|---|---|---|---|
| Typography | `bodySmall` | `.caption` | description が `.subheadline` から一段下げる、Compose と同じ視覚的ヒエラルキー |
| Color | `onSurfaceVariant` | `.secondary` | description と同じ secondary tier (Compose も description と details で同色) |
| Max lines | `maxLines = 2` + `Ellipsis` | `.lineLimit(1...2)` + `.minimumScaleFactor(0.7)` + `.allowsTightening(true)` | X1 description precedent (Dynamic Type AX5 clamp) を継承、加えて記号列であるため `.allowsTightening(true)` (DiscoveryScreen.swift details line と同じ) |
| Null handling | `pattern.gauge?.let` | `pattern.gauge.map` | DiscoveryScreen.swift L433-441 と同じ pattern、Compose bug-for-bug parity (empty string でも `"Gauge: "` を出す挙動) |

### 重要 review 観点

- **DiscoveryScreen との iOS 内部 consistency** を維持 — 同フィールド (gauge/yarnInfo/needleSize) の表示ロジックを DiscoveryScreen.buildDetails() の既存パターンと完全一致させた
- **Parametric localization** は Phase 33.1.7 precedent (`NSLocalizedString + String(format:)`) を遵守 — SwiftUI `LocalizedStringKey` literal-promotion-table は format args を取れない既知制約
- **i18n.tsv 不要** — `label_gauge_value` / `label_yarn_value` / `label_needle_value` キーは xcstrings に既存 (Phase 1 agent 3 で確認、DiscoveryScreen でも既使用)
- **新規 test なし** — declared 通り visual-only display polish (振る舞いの変化なし、新入力経路なし、PatternRow 専用 XCUITest fixture infra 未整備のため新規追加は scope expansion)
