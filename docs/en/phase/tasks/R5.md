### M1 — OnboardingScreen page indicator (WCAG 1.4.1 + 4.1.2)
`PageIndicator` (`OnboardingScreen.kt`) は active/inactive を `primary` vs `outlineVariant` の色のみで識別していた。修正 (a): active dot に `border(1.5.dp, onSurface, CircleShape)` を加算 — 色弱でも識別可能なリング、size は 10.dp 維持 (dot は touch ターゲットではないため WCAG 2.5.5 影響なし)。修正 (b): parent Row に `semantics { contentDescription = "Page X of Y"; liveRegion = LiveRegionMode.Polite }` を追加 — pager swipe ごとに SR が politely 発話。bilingual fallback inline (`if (isJa) "${pageCount}ページ中${currentPage + 1}ページ目" else "Page ${currentPage + 1} of $pageCount"`)、新規キー `a11y_state_page_indicator_x_of_y` を `R5.i18n.tsv` で出荷。

### M3 — MfaEnrollment recovery-code copy affordance (WCAG 4.1.2)
`RecoveryCodeContent` (`MfaEnrollmentScreen.kt`) の `Modifier.clickable { ... }` は role/onClickLabel/SR feedback なし。`Modifier.clickable(onClickLabel = "Copy recovery code", role = Role.Button) { ... }` に変更 → TalkBack/VoiceOver が「ダブルタップでコピー」を発話。コピー成功時に `copiedAnnouncement: MutableState<String>` を更新、`Modifier.semantics { liveRegion = LiveRegionMode.Polite }` 付き Text で polite SR announcement。bilingual fallback inline、新規キー `a11y_action_copy_recovery_code` + `state_recovery_code_copied` を `R5.i18n.tsv` で出荷。`clipboard.setText` 動作は変更せず既存挙動保護。

### M5 — iOS ChartConflictResolution selected trait (WCAG 1.4.1 + 4.1.2)
`pickerButton` (`ChartConflictResolutionScreen.swift`) の選択状態は `.borderedProminent` vs `.bordered` の色のみ。`.accessibilityAddTraits(isSelected ? .isSelected : [])` を Button modifier chain に追加 → VoiceOver が「選択中」trait を自動翻訳・発話 (OS-managed)。

### M6 — iOS decorative icon `.accessibilityHidden(true)` × 3 サイト (WCAG 4.1.2)
SF Symbol 名 ("g.circle.fill" / "house" / "plus.circle" 等) が adjacent text の前に VoiceOver で読み上げられていた。各サイトで隣接 Text が意味を担うため装飾扱いで安全に hide:
- `LoginScreen.swift:126` Google "g.circle.fill" — adjacent "Continue with Google" Text が意味を保持
- `OnboardingScreen.swift:128` ヒーローアイコン (3 通常ページ) + L174 DiagnosticConsentPageView ヒーロー — adjacent title/body Text が意味を保持
- `ActivityFeedScreen.swift:52` activity type icon — adjacent `headline` が "Project started by X" / "Pattern shared by X" 等の意味を保持

### M7 — LinearProgressIndicator `progressBarRangeInfo` × 2 サイト (WCAG 4.1.2)
`ProjectListScreen.kt:660-663` (`ProjectCard`) + `ProjectDetailScreen.kt:670-684` (`CounterSection`) の LinearProgressIndicator は SR が "progress bar" としか発話せず、隣接 Text は別要素として扱われていた。`Modifier.semantics { progressBarRangeInfo = ProgressBarRangeInfo(current = currentRow.toFloat(), range = 0f..totalRows.toFloat()); contentDescription = "X of Y rows completed" }` を追加 → SR が「5 of 30 rows completed」のように value + label を発話。bilingual fallback inline、新規キー `a11y_progress_rows_completed_x_of_y` を `R5.i18n.tsv` で出荷。pattern source: `ChartEditorScreen.kt:1207` の R3 progressBarRangeInfo 先行実装。

### R3 Follow-up #2 — iOS EmailConfirmationSentView heading (WCAG 1.3.1)
`LoginScreen.swift:289-292` の `.title2` は VoiceOver headings rotor から漏れていた。`.accessibilityAddTraits(.isHeader)` を 1 行追加 → 同ファイル L351-353 `LinkIdentityForm` の `.title2` 既存実装と parity 達成。これで iOS `.title2` heading は 3 サイト全て `.isHeader` 適用済み。

### R3 Follow-up #4 — PaywallScreen PackageRow selection (WCAG 1.4.1 + 4.1.2)
`PackageRow` (`PaywallScreen.kt:342-405`) は selected を `containerColor` の色変化のみで表現、SR から「ラジオボタン」trait + selected/unselected state がともに不可視。`Modifier.clickable(role = Role.RadioButton, onClick = onSelect).semantics { this.selected = selected; stateDescription = if (selected) "Selected" else "Not selected" }` に変更 → SR が「Annual subscription option, radio button, selected」のように発話。bilingual fallback inline、新規キー `state_selected` + `state_not_selected` を `R5.i18n.tsv` で出荷。pattern source: `BiometricSettingsScreen.kt` の R1c `Role.RadioButton` 先行実装。

### R4 Follow-up #4 — ProjectDetail fixed-dp button-content icons → scalable (WCAG 1.4.4)
`ProjectDetailScreen.kt` の Button/TextButton 内 decorative leading-icon glyphs が fixed dp で fontScale 追従しなかった:
- `StatusToggleButton` L896 Refresh 18.dp + L907 CheckCircle 18.dp → `Modifier.size(with(LocalDensity.current) { 18.sp.toDp() })`
- `ResetProgressButton` L933 Refresh 18.dp → 同上
- `AddNoteDialog` photo-attached row L982 Check 16.dp + add-photo TextButton L1009 Add 16.dp → `Modifier.size(with(LocalDensity.current) { 16.sp.toDp() })`

`.sp.toDp()` は `LocalDensity.current` を介して fontScale を factor in、Dynamic Type Largest accessibility size でアイコンが text label と同比率で拡大。**意図的に scope 外** (out-of-scope justification 残す): L997 IconButton (size=24.dp) 内側の Close (size=16.dp) — IconButton そのものが touch target で WCAG 2.5.5 24.dp floor を維持、scaling すると Largest 48.dp Close glyph が 24.dp IconButton 内側にハミ出して broken UI、 touch-target redesign は R5 scope 外。L1021 `CircularProgressIndicator(size(16.dp))` は Icon ではなく spinner、scaling は不要。
